# 12 - Supabase: Definicion y Replicacion de Base de Datos

## Objetivo

Este documento describe la base de datos Supabase requerida por PharmaOps 360 y deja un "super comando" reproducible para levantar el esquema base en un proyecto nuevo.

El SQL esta disenado como baseline compatible con la aplicacion actual. Es idempotente en la mayoria de objetos (`create table if not exists`, `add column if not exists`, `drop policy if exists`) para poder ejecutarlo durante una instalacion o recuperacion controlada.

## Super Comando

### Opcion A - SQL Editor de Supabase

1. Entrar al proyecto Supabase.
2. Ir a `SQL Editor`.
3. Pegar completo el bloque de la seccion [SQL completo replicable](#sql-completo-replicable).
4. Ejecutar.
5. Validar que existan las tablas, buckets y politicas.

### Opcion B - CLI con `psql`

Guardar el bloque SQL como `supabase-pharmaops-bootstrap.sql` y ejecutar:

```bash
psql "$SUPABASE_DB_URL" -v ON_ERROR_STOP=1 -f supabase-pharmaops-bootstrap.sql
```

Variables esperadas:

- `SUPABASE_DB_URL`: cadena PostgreSQL del proyecto Supabase.
- El usuario debe tener permisos para crear tablas, politicas RLS y buckets de Storage.

## Modulos Cubiertos

- IAM y perfiles: `usuarios_roles`
- Inventario regulado: `activos`
- Historial de activos: `activos_historial_cambios`
- Ordenes / RUI: `mantenimientos_registros`
- Plantillas dinamicas: `mantenimiento_plantillas_campos`
- Formularios EAV historicos: `formularios_campos`, `formularios_respuestas`
- Firmas y trazabilidad: `mantenimiento_firmas`, `audit_trail`, `auditoria_log_cambios`
- Perimetro de seguridad: `access_denied_logs`
- Storage: `evidencias-mantenimiento`, `activos`

## Relaciones Principales

| Origen | Destino | Uso |
| --- | --- | --- |
| `activos.asset_code` | `mantenimientos_registros.asset_code` | Une el activo con sus ordenes y RUI. |
| `mantenimientos_registros.id` | `formularios_respuestas.mantenimiento_id` | Respuestas EAV por inspeccion. |
| `formularios_campos.id` | `formularios_respuestas.campo_id` | Metadata del campo respondido. |
| `activos.asset_type` | `mantenimiento_plantillas_campos.asset_type` | Plantilla dinamica por tipo de activo. |
| `activos.uuid` | `activos_historial_cambios.activo_id` | Historial GxP de cambios del activo. |

## Estados Documentales Relevantes

La aplicacion usa estos estados en `mantenimientos_registros.status`:

- `draft`
- `PENDING_TECHNICIAN`
- `PENDING_SUPERVISOR`
- `PENDING_QUALITY`
- `PENDING_MANAGEMENT`
- `approved`
- `rejected`
- `RECHAZADO_TECNICO`

## Capacidades IAM Relevantes

La aplicacion no depende solo de `role`. Para zonas criticas usa flags booleanos:

- `can_create_assets`
- `can_execute_maintenance`
- `can_review`
- `can_approve`
- `can_view_audit`
- `can_manage_users`

La ruta `Crear Ordenes` permite acceso si el usuario activo tiene:

```txt
can_approve = true OR can_create_assets = true
```

## SQL completo replicable

```sql
-- PharmaOps 360 - Supabase bootstrap
-- Baseline compatible con la aplicacion actual.
-- Ejecutar en Supabase SQL Editor o con psql.

begin;

create extension if not exists "pgcrypto";

-- =========================================================
-- 1. Usuarios, perfiles y capacidades
-- =========================================================

create table if not exists public.usuarios_roles (
  id bigint generated by default as identity primary key,
  user_email text not null,
  full_name text,
  job_title text,
  role text not null default 'Tecnico',
  user_type text,
  site text,
  area text,
  active boolean not null default true,
  can_create_assets boolean not null default false,
  can_execute_maintenance boolean not null default false,
  can_review boolean not null default false,
  can_approve boolean not null default false,
  can_view_audit boolean not null default false,
  can_manage_users boolean not null default false,
  requires_2fa boolean not null default false,
  notes text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_usuarios_roles_email
  on public.usuarios_roles (lower(user_email));

create index if not exists idx_usuarios_roles_active
  on public.usuarios_roles (active);

-- =========================================================
-- 2. Activos regulados
-- =========================================================

create table if not exists public.activos (
  id bigint generated by default as identity primary key,
  uuid uuid not null default gen_random_uuid(),
  asset_code text not null,
  asset_name text not null,
  asset_type text not null default 'Sistemas HVAC',
  site text,
  area text not null default 'Sin area',
  location_detail text,
  brand text,
  model text,
  serial_number text,
  capacity text,
  capacity_unit text,
  installation_date date,
  status text not null default 'Operativo',
  maintenance_frequency text not null default 'Mensual',
  last_maintenance_date date,
  next_maintenance_date date,
  internal_responsible text,
  technical_provider text,
  qr_possible boolean not null default false,
  notes text,
  imagen_url text,
  disabled_at timestamptz,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint activos_uuid_key unique (uuid),
  constraint activos_asset_code_key unique (asset_code)
);

create index if not exists idx_activos_asset_type
  on public.activos (asset_type);

create index if not exists idx_activos_asset_code
  on public.activos (asset_code);

create index if not exists idx_activos_status
  on public.activos (status);

-- =========================================================
-- 3. Ordenes de mantenimiento / RUI
-- =========================================================

create table if not exists public.mantenimientos_registros (
  id bigint generated by default as identity primary key,
  uuid uuid not null default gen_random_uuid(),
  record_code text,
  asset_code text not null,
  template_code text,
  assigned_technician text,
  scheduled_date timestamptz,
  executed_at timestamptz,
  status text not null default 'draft',
  notes text,
  supervisor_signed_by text,
  supervisor_signed_at timestamptz,
  quality_signed_by text,
  quality_signed_at timestamptz,
  management_signed_by text,
  management_signed_at timestamptz,
  rejection_comments text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint mantenimientos_registros_uuid_key unique (uuid),
  constraint fk_mantenimientos_registros_activos
    foreign key (asset_code)
    references public.activos (asset_code)
    on update cascade
    on delete restrict
);

create index if not exists idx_mantenimientos_asset_code
  on public.mantenimientos_registros (asset_code);

create index if not exists idx_mantenimientos_status
  on public.mantenimientos_registros (status);

create index if not exists idx_mantenimientos_executed_at
  on public.mantenimientos_registros (executed_at desc);

-- =========================================================
-- 4. Plantillas dinamicas por tipo de activo
-- =========================================================

create table if not exists public.mantenimiento_plantillas_campos (
  id bigint generated by default as identity primary key,
  asset_type text not null,
  seccion_nombre text not null,
  campo_nombre text not null,
  tipo_dato text not null check (tipo_dato in ('OK_NOK', 'NUMERIC', 'TEXT')),
  unidad_medida text not null default 'N/A',
  field_key text,
  section_name text,
  field_label text,
  field_type text,
  unit text,
  required boolean not null default true,
  options jsonb,
  section_order integer not null default 999,
  field_order integer not null default 999,
  created_at timestamptz not null default now()
);

create index if not exists idx_plantillas_asset_type
  on public.mantenimiento_plantillas_campos (asset_type);

create index if not exists idx_plantillas_order
  on public.mantenimiento_plantillas_campos (asset_type, section_order, field_order);

-- =========================================================
-- 5. Formularios historicos EAV
-- =========================================================

create table if not exists public.formularios_campos (
  id bigint generated by default as identity primary key,
  field_key text not null,
  section_name text,
  field_label text not null,
  field_type text not null default 'TEXT',
  unit text,
  evidence_required boolean not null default false,
  required boolean not null default false,
  section_order integer not null default 999,
  field_order integer not null default 999,
  created_at timestamptz not null default now(),
  constraint formularios_campos_field_key_key unique (field_key)
);

create table if not exists public.formularios_respuestas (
  id bigint generated by default as identity primary key,
  mantenimiento_id bigint not null,
  campo_id bigint not null,
  valor_texto text,
  valor_seleccion text,
  valor_numerico numeric,
  valor_booleano boolean,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint fk_formularios_respuestas_mantenimiento
    foreign key (mantenimiento_id)
    references public.mantenimientos_registros (id)
    on update cascade
    on delete cascade,
  constraint fk_formularios_respuestas_campo
    foreign key (campo_id)
    references public.formularios_campos (id)
    on update cascade
    on delete restrict,
  constraint formularios_respuestas_unique_mantenimiento_campo
    unique (mantenimiento_id, campo_id)
);

create index if not exists idx_formularios_respuestas_mantenimiento
  on public.formularios_respuestas (mantenimiento_id);

-- Campo virtual usado cuando la plantilla no trae un field_type evidence.
insert into public.formularios_campos (
  field_key,
  section_name,
  field_label,
  field_type,
  evidence_required,
  required,
  section_order,
  field_order
)
values (
  'evidencias_hvac',
  'Evidencias',
  'Evidencias HVAC',
  'evidence',
  true,
  false,
  999,
  999
)
on conflict (field_key) do nothing;

-- =========================================================
-- 6. Firmas, auditoria y eventos GxP
-- =========================================================

create table if not exists public.mantenimiento_firmas (
  id bigint generated by default as identity primary key,
  mantenimiento_uuid uuid not null,
  accion text not null,
  usuario text not null,
  timestamp timestamptz not null default now(),
  comentarios text,
  created_at timestamptz not null default now()
);

create table if not exists public.audit_trail (
  id bigint generated by default as identity primary key,
  entity text not null,
  entity_uuid uuid,
  accion text not null,
  usuario text,
  timestamp timestamptz not null default now(),
  comentarios text,
  created_at timestamptz not null default now()
);

create table if not exists public.auditoria_log_cambios (
  id bigint generated by default as identity primary key,
  usuario_email text,
  usuario_id uuid,
  timestamp timestamptz not null default now(),
  accion text not null,
  justificacion text,
  payload_snapshot jsonb,
  entidad text,
  entidad_uuid text,
  created_at timestamptz not null default now()
);

create index if not exists idx_auditoria_entidad_uuid
  on public.auditoria_log_cambios (entidad, entidad_uuid);

create table if not exists public.activos_historial_cambios (
  id uuid primary key default gen_random_uuid(),
  activo_id uuid not null,
  usuario_id uuid not null,
  accion text not null check (accion in ('insert', 'update', 'inactivate')),
  valores_anteriores jsonb,
  valores_nuevos jsonb,
  justificacion_gxp text not null,
  creado_at timestamptz not null default now(),
  constraint fk_activos_historial_cambios_activo_uuid
    foreign key (activo_id)
    references public.activos (uuid)
    on update restrict
    on delete restrict
);

-- =========================================================
-- 7. Seguridad perimetral
-- =========================================================

create table if not exists public.access_denied_logs (
  id bigint generated by default as identity primary key,
  user_id uuid,
  user_role text,
  target_url text not null,
  server_utc_timestamp timestamptz not null default now(),
  created_at timestamptz not null default now()
);

create index if not exists idx_access_denied_logs_profile_target
  on public.access_denied_logs (user_role, target_url);

-- =========================================================
-- 8. Storage buckets
-- =========================================================

insert into storage.buckets (id, name, public)
values
  ('evidencias-mantenimiento', 'evidencias-mantenimiento', true),
  ('activos', 'activos', false)
on conflict (id) do nothing;

-- 9. RLS baseline
-- =========================================================

alter table public.activos enable row level security;
alter table public.mantenimientos_registros enable row level security;
alter table public.mantenimiento_plantillas_campos enable row level security;

alter table public.formularios_respuestas enable row level security;
```

## Validacion Posterior

Ejecutar estas consultas despues del bootstrap:

```sql
select table_name
from information_schema.tables
where table_schema = 'public'
  and table_name in (
    'usuarios_roles',
    'activos',
    'mantenimientos_registros',
    'mantenimiento_plantillas_campos',
    'formularios_campos',
    'formularios_respuestas',
    'mantenimiento_firmas',
    'audit_trail',
    'auditoria_log_cambios',
    'activos_historial_cambios',
    'access_denied_logs'
  )
order by table_name;

select id, name, public
from storage.buckets
where id in ('evidencias-mantenimiento', 'activos')
order by id;

select asset_code, asset_name, asset_type, status
from public.activos
order by asset_code;
```

## Notas de Compliance

- No se define `delete` operativo para activos. La aplicacion usa baja logica con `status` y `disabled_at`.
- `activos_historial_cambios` queda sin politicas `update` o `delete`; con RLS activo, esas operaciones quedan bloqueadas salvo service role.
- `auditoria_log_cambios`, `audit_trail` y `mantenimiento_firmas` deben tratarse como registros historicos.
- Para produccion regulada, las politicas amplias de escritura deben reemplazarse por funciones de autorizacion basadas en `usuarios_roles`.
